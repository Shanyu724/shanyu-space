# 🏠 山雨·个人站 — Obsidian 使用指南

> 这是你的网站在 Obsidian 里的样子。整个仓库就是一个 Vault，**你在这里写的任何 `.md` 文件，都可能成为网站上的一篇文章**。

## 📂 文件夹约定

| 文件夹 | 用途 | 写完会出现在 |
|--------|------|------------|
| `content/blog/geo/` | 地缘观察 | 网站 `/blog/geo` |
| `content/blog/finance/` | 金融制度 | 网站 `/blog/finance` |
| `content/blog/study/` | 学习笔记 | 网站 `/blog/study` |
| `content/blog/essays/` | 随笔 | 网站 `/blog/essays` |
| `content/portfolio/projects.json` | 作品集 | 网站 `/portfolio` |
| `content/workshop/tools.json` | 工坊（AI 工具） | 网站 `/workshop` |
| `content/assets/` | 图片、附件 | 通过 Markdown 链接引用 |

⚠️ 其他文件（如 `src/`、`package.json`）是网站代码，**不要改**，除非你知道自己在做什么。

## ✍️ 发文章的标准流程

### 1. 新建文件

在 Obsidian 左侧文件树，右键 `content/blog/essays/` → **新建文件** → 起个名字（如 `第一次去杭州.md`）。

### 2. 写 frontmatter

每篇博客**第一行必须是**：

```yaml
---
title: "文章标题"
date: "2026-06-26"
description: "一两句话介绍，会显示在卡片上"
tags: [旅行, 随笔]
published: true
---

从这里开始写正文…
```

字段说明：
- `title` — 文章标题（必填）
- `date` — 发布日期，格式 YYYY-MM-DD（必填）
- `description` — 简介（建议填）
- `tags` — 标签，可以多个
- `published` — 设为 `false` 可以先写成草稿、不发布

### 3. 写正文

正常用 Markdown 写就行。常用语法：

```markdown
## 二级标题
**加粗** *斜体* [链接](https://example.com)
- 列表项
1. 有序列表
> 引用
![图片描述](content/assets/xxx.png)
```

### 4. 保存 + 发布

**方法 A — Obsidian Git 插件（推荐，一次设置永久自动）**：
- 安装插件后，每次你切换窗口/关闭 Obsidian，**它自动提交并推送到 GitHub**
- 几分钟后网站就更新了

**方法 B — 手动**：
- 按 `Ctrl + P`，输入 `Git: Commit`，回车
- 再输入 `Git: Push`，回车

## ❓ 常见问题

**Q: 我把网站代码（`src/`、`package.json`）误删了怎么办？**
A: 没事，所有改动都在 GitHub。你删的是本地文件，重新从 GitHub clone 就行。

**Q: 我不想用 GitHub 直接，能不能只写不发？**
A: 把 `published` 设为 `false` 即可。这篇文章不会出现在网站上，但还在你的笔记里。

**Q: 文章里有图片怎么办？**
A: 把图片复制到 `content/assets/` 文件夹（Obsidian 设置里已经配了自动归档到这个位置），然后用 `![描述](image.png)` 引用。

**Q: 我想加一个新分类（比如"摄影"）？**
A: 在 `content/blog/` 下新建一个文件夹（比如 `photography`），在里面写 markdown 就行。但需要让代码知道这个分类——告诉 Claude 帮你加。
